One-Click Deploy

Deployment Flow

From API key to running agent in under 60 seconds. Five automated steps with real-time log streaming.

1

Configure

Generate .env with auto-assigned PORT (10000–65000) and gateway token. User provides API keys and optional Telegram/WhatsApp config.

2

Pull Image

Pull the latest ghcr.io/openclaw/openclaw Docker image. Progress is streamed to the UI terminal in real time.

3

Start Container

Run docker compose up -d in an isolated deployments/<id>/ directory. Container generates openclaw.json dynamically from env vars.

4

Gateway Auth

HTTP health probe + full WebSocket handshake (OpenClaw protocol v3) to verify the gateway is accessible and authenticated.

5

Chat Ready

Connect to the deployed agent via RemoteJasonClient with client.id="cli". Agent is online and ready for conversation.

Getting Started

Quick Setup

Get Aether Orchestrator running locally in under 5 minutes.

Clone & Backend

Set up the Python virtual environment and install dependencies.

git clone git@github.com:bhavith-oc/Agent-orchestrator.git
cd Agent-orchestrator/api
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# Edit .env — set your OPENROUTER_API_KEY
python3 main.py

Frontend

Install Node dependencies and start the Vite dev server.

cd ui
npm install
npm run dev
# Open http://localhost:5173

Deploy an Agent

Follow the onboarding flow in the UI, or use the CLI script.

# Via UI: open http://localhost:5173 → follow onboarding

# Via CLI:
./deploy.sh

# Via API:
curl -X POST http://localhost:8000/api/deploy/configure \
  -H "Content-Type: application/json" \
  -d '{"openrouter_api_key": "sk-or-v1-..."}'

Environment Variables

Key configuration options for backend and frontend.

# api/.env (required)
OPENROUTER_API_KEY=sk-or-v1-your-key
SECRET_KEY=change-in-production

# api/.env (optional — auto-connect to remote OpenClaw)
REMOTE_JASON_URL=ws://your-server:61816
REMOTE_JASON_TOKEN=your-gateway-token

# ui/.env (optional)
VITE_GOOGLE_CLIENT_ID=your-google-client-id
VITE_LEGACY_LOGIN=false
